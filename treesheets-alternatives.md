# Top 10 GitHub Repositories with TreeSheets-like Capabilities

This document presents the top 10 GitHub repositories that offer capabilities similar to TreeSheets, specifically focusing on features like:
- **Tags** - Organizing and categorizing information
- **Hierarchy Swap** - Ability to reorganize and pivot hierarchical data
- **Keyboard Shortcuts** - Efficient navigation and manipulation
- **Tree/Grid-based Data Organization** - Hierarchical data structures with visual representation
- **Link Traversal** - Navigation between related items

---

## 1. Logseq
**Repository:** [logseq/logseq](https://github.com/logseq/logseq)  
**Stars:** 39,359 | **Language:** Clojure/ClojureScript | **Status:** Active

### Key Features Similar to TreeSheets:
- **Hierarchical outliner** with block-based organization (similar to tree structure)
- **Tagging system** with rich metadata support
- **Bi-directional links** (similar to F6 link traversal in TreeSheets)
- **Keyboard-first navigation** with extensive shortcuts
- **Graph view** for visualizing relationships between notes
- **Local-first, privacy-focused** with optional sync

### Why It's Similar:
Logseq provides a hierarchical outliner approach where you can organize information in nested blocks. While it uses a graph structure rather than a strict tree, it offers powerful tagging, keyboard shortcuts, and the ability to traverse links between notes. The query language allows filtering and reorganizing data similar to hierarchy swap.

## 2. Trilium Notes

**Stars:** 32,861 | **Language:** TypeScript | **Status:** Active (TriliumNext fork)

- **Keyboard shortcuts** for navigation and manipulation

- **Link management** with back-references

### Why It's Similar:
Trilium Notes is one of the closest alternatives to TreeSheets in terms of explicit tree structure. It has a rich tagging system, keyboard-driven interface, and the ability to reorganize information through cloning and relationships. The scriptable nature allows for custom hierarchy manipulations.

## Selection Criteria

These repositories were selected based on:

1. **Hierarchical Data Organization**: Support for tree or nested structures
2. **Tagging/Metadata Systems**: Ability to categorize and organize information
3. **Data Reorganization**: Features similar to TreeSheets' hierarchy swap (pivoting, filtering, querying)
4. **Keyboard Shortcuts**: Efficient keyboard-driven navigation
5. **Link Management**: Ability to traverse connections between items
6. **Active Development**: Projects that are actively maintained
7. **Community Adoption**: GitHub stars as an indicator of usage and quality
8. **Open Source**: All projects are open source and available on GitHub

---

## Conclusion

While TreeSheets' unique combination of grid-based UI, hierarchy swap, and free-form data organization is rare, these ten repositories offer similar capabilities in different forms:

- **For pure tree/outline lovers**: Logseq, Trilium, TreeHouse
- **For spreadsheet-style organization**: Luckysheet/Univer, Tad
- **For mind mapping**: Freeplane
- **For note-taking with hierarchy**: Joplin, SiYuan
- **For developers building similar tools**: react-sortable-tree
- **For Obsidian users**: Dataview plugin

Each project brings its own philosophy and implementation to hierarchical data management, but all share the core principles that make TreeSheets powerful: hierarchical organization, flexible reorganization, keyboard efficiency, and structured yet flexible data management.
